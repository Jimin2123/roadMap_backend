# 커밋 메세지 작성 예시

## 작성 예시
```bash
Feat(Member): 회원가입 API 구현

- 회원가입 시 이메일 중복 체크 기능 추가
- 회원가입 시 비밀번호 암호화 기능 추가
- 회원가입 시 이메일 인증 기능 추가
- 회원가입 시 닉네임 중복 체크 기능 추가

관련 이슈: #123
```

## Header 제목 영역
1. 제목은 50자 이내로 작성
2. 제목 끝에는 마침표 금지
3. 영어일 경우 제목 첫 글자는 대문자

### 타입 형식

```
- [Feat] : 새로운 기능 추가
- [Fix] : 버그 수정
- [Docs] : 문서 수정
- [Refactor] : 코드 리팩토링
- [Style] : 코드 포맷팅, 세미콜론 누락 (코드 변경 X)
- [Chore] : 빌드 업무 수정, 패키지 매니저 수정 (코드 변경 X)
```

## Body 본문 영역

- 변경 사항에 대한 상세 설명을 작성
- 본문 각 행은 72자 이내로 작성
- "무엇이"와 "왜"를 설명하고, "어떻게"는 필요시 간단히 작성
- 예: JWT를 사용한 인증 로직 추가 및 테스트 코드 작성

## Footer 푸터 영역
- 관련 이슈 번호를 작성
- 예: 관련 이슈: #123
